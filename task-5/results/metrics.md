# Выбранные метрики (стандартные, без кастомных)

В рамках POC используются стандартные метрики Spring Boot Actuator + Micrometer, доступные в формате Prometheus на `/actuator/prometheus`.
В этом задании не требуется разрабатывать бизнес-метрики, поэтому достаточно стандартных показателей, чтобы продемонстрировать:

- доступность приложения,
- нагрузку по ресурсам,
- состояние JVM,
- готовность приложения,
- возможность строить дашборды и оповещения.

## Ключевые метрики

### Доступность (базовый индикатор)

- `up{job="batch-processing"}`
  - Источник: результат scrape Prometheus.
  - Зачем: самый простой сигнал, что приложение доступно и endpoint метрик работает.

### CPU приложения

- `process_cpu_usage`
  - Зачем: показывает нагрузку CPU и помогает ловить перегруз/нехватку ресурсов.

### Память (JVM heap)

- `jvm_memory_used_bytes{area="heap"}`
  - Зачем: контроль потребления памяти, косвенная диагностика утечек и sizing.

### Потоки JVM

- `jvm_threads_live_threads`
  - Зачем: помогает выявлять “разъезд” по потокам (thread leak) и аномальную конкуррентность.

### Время готовности приложения

- `application_ready_time_seconds`
  - Зачем: показывает скорость старта/готовности (важно для деплоев и масштабирования).

## Почему стандартных метрик достаточно

- Цель задания — показать работающий контур мониторинга (scrape → dashboard → alert), а не глубокую доменную наблюдаемость.
- Инциденты уровня “приложение не отвечает/ресурсы кончаются” покрываются стандартными JVM/process метриками.
- В будущем можно добавить доменные метрики (например, количество обработанных строк, число ошибок валидации), но это выходит за рамки текущего POC.
